```javascript

```

Declaring variable names directly without using let, var or const; This method is used to update let variable and var variable.</i>

###### 2. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
function sum(a, b) {
  return a + b;
}

sum(1, "2");
```

- A: `NaN`
- B: `TypeError`
- C: `"12"`
- D: `3`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: C

<i>If one of the operands is a number and the other is a string, JavaScript will convert the number to a string and perform string concatenation. Then they will be concatenated together, not added mathematically.</i>

</p>
</details>

###### 3. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
const food = ["🍕", "🍫", "🥑", "🍔"];
const info = { favoriteFood: food[0] };

info.favoriteFood = "🍝";

console.log(food);
```

- A: `['🍕', '🍫', '🥑', '🍔']`
- B: `['🍝', '🍫', '🥑', '🍔']`
- C: `['🍝', '🍕', '🍫', '🥑', '🍔']`
- D: `ReferenceError`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: A

<i>There have an array <b>food</b> and an object info with a property <b>favoriteFood</b> assigned to the first element of the <b>food</b> array. Aassigning a new value "🍝" to <b>info.favoriteFood</b>, this change does not affect the original <b>food</b> array.
So, console.log(food), it will output the original array, which remains unchanged</i>

</p>
</details>

###### 4. Write the `correct answer` from the following options and give an explanation (2-5 lines).


- A: `Hi there,`
- B: `Hi there, undefined`

- D: `ReferenceError`

<p>

#### Answer: B

<i>In this case, "undefined" is the value of the <b>name</b> parameter because any value didn't provide there when calling the function. Resulting the string "Hi there, " is concatenated with the undefined value.</i>

</p>
